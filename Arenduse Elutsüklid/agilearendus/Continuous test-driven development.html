CTDD

Pidev testipõhine arendus (CTDD) on tarkvaraarenduse tava, mis laiendab testipõhist arendust (TDD) automaatse testide teostamise abil taustal, mida mõnikord nimetatakse pidevaks testimiseks.

  1. Arendaja kirjutab esmalt väikese testi, mis ebaõnnestub, kuna funktsionaalsus pole veel olemas.
  2. Koodi kirjutatakse minimaalselt nii palju, et test saaks edukalt läbi minna.
  3. Automaatne testimissüsteem käivitab testid pidevalt ja annab kohe tagasisidet koodi seisukorra kohta.
  4. Vead või regressioonid avastatakse varakult ja neid parandatakse kohe, mis vähendab defektide kuhjumist.

CASE

GitHub
GitHub Actions võimaldab sul seadistada automaatseid töövooge, mis jooksutavad teste iga kord, kui kood muutub (push, pull request vms).
